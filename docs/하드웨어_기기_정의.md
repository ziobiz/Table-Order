# 하드웨어·기기 정의

본사·가맹점·배달기사·고객별로 **어떤 기기에서 운영되는지** 정의합니다. UI·반응형·터치 지원은 이 정의를 기준으로 합니다.

---

## 1. 본사 (HQ)

| 항목 | 정의 |
|------|------|
| **주 사용 기기** | PC (Windows)에서 작업 |
| **추가 지원** | iPad, Android 패드에서도 작업 가능해야 함 |
| **요구사항** | PC에서의 전체 기능이 패드에서도 동작. 화면 크기에 맞춰 레이아웃·버튼 크기 조정 (반응형). |

- 본사 대시보드, 가맹점 관리, 정책, 배달 API, 가맹점 신청 승인 등 **모든 본사 페이지**는 PC + 패드(iPad/Android 패드)에서 100% 운용 가능하도록 구현·검증합니다.

---

## 2. 가맹점 (Store)

| 항목 | 정의 |
|------|------|
| **설정·세팅** | PC 또는 패드에서 가능 (본사와 동일) |
| **일상 운영** | **패드(태블릿)에서 운영** |
| **요구사항** | **UI 및 내용이 패드에서 100% 운용이 되도록 지원**해야 함. 터치 조작, 패드 해상도, 가독성·버튼 크기를 패드 기준으로 최적화. |

- POS, 주문/주방 화면, 배차 대시보드, 배달 기사 관리, 배달앱 연동, QR 관리, 정산 등 **가맹점 운영에 쓰는 모든 화면**은 **패드에서만 사용한다고 가정**하고, 패드에서 100% 완결되도록 설계·구현합니다.
- 세팅(예: KDS 테마, 알림 시간 등)은 PC에서 해도 되고 패드에서 해도 되며, 동일한 UI로 제공합니다.

---

## 3. 배달기사 (Rider / Deliver)

| 항목 | 정의 |
|------|------|
| **운영 기기** | **휴대폰(스마트폰)** |
| **요구사항** | 기사용 모든 화면(대기 목록, 수락, 내 배달, 픽업/배달 완료 등)이 **휴대폰에서 100% 운용**되도록 지원. 모바일 퍼스트, 터치 친화, 작은 화면에서 가독성·버튼 크기 확보. |

- 기사 로그인, 대기 중인 배달 목록, 콜 수락, 내 배달, 픽업 완료·배달 완료 등 **배달기사가 쓰는 모든 UI**는 **휴대폰 기준**으로 설계·구현·테스트합니다.

---

## 4. 고객 (Customer)

| 항목 | 정의 |
|------|------|
| **운영 기기** | **휴대폰, 패드, PC 모든 기종** |
| **요구사항** | 주문 방식 선택, 메뉴·장바구니, 주문 확인·결제·완료 등 **고객이 접하는 모든 화면**이 휴대폰·패드·PC에서 모두 정상 동작. 반응형으로 뷰포트에 맞춰 레이아웃·폰트·버튼 크기 조정. |

- index, menu, menu_detail, order_review, order_complete, 리뷰 등 **고객용 페이지 전체**는 **모든 기기(휴대폰·패드·PC)** 에서 100% 운용 가능하도록 구현·검증합니다.

---

## 5. 구현 시 적용 사항 (요약)

| 대상 | 기기 | 적용 시 유의사항 |
|------|------|------------------|
| 본사 | PC, 패드 | viewport 메타 태그, 중소형 브레이크포인트에서도 레이아웃·버튼이 깨지지 않도록. 패드 터치 시 버튼/링크 터치 영역 충분히. |
| 가맹점 | **패드 중심** | 패드 해상도(768px~1024px 등) 기준 레이아웃·폰트·버튼 크기. 터치 타겟 최소 44px 권장. 패드에서만 쓴다고 가정하고 100% 완결. |
| 배달기사 | **휴대폰** | 모바일 퍼스트. 뷰포트 meta, 작은 화면에서 한 손 조작·가독성. 버튼·리스트 터치 영역 넉넉히. |
| 고객 | 휴대폰·패드·PC | 반응형(모바일 → 태블릿 → 데스크톱). 미디어 쿼리·flex/grid로 모든 뷰포트에서 레이아웃·내용·CTA 노출. |

- **공통**: `viewport` meta 태그 설정, 터치 가능 기기에서 호버 대신 터치·포커스로 동작 검증.
- **문서 위치**: 이 정의는 `docs/하드웨어_기기_정의.md` 에 있으며, UI·반응형·신규 페이지 설계 시 이 정의를 따릅니다.
