## 2. 업계 표준 POS 기능 (TouchBistro·업계 참고)

### 2-1. Front of House
- **Customer Facing Display**: 카운터용 고객 대면 디스플레이
- **Payments**: 현금, 카드, 디지털 지갑, **선물권(기프트카드)**
- **POS**: 주문·결제, 테이블사이드 주문·결제, **체크 분할(split by item/guests)**

### 2-2. Back of House
- **Profit Management**: 재고·매출·비용 통합 백오피스
- **KDS**: 주방 디스플레이
- **Labor Management**: 직원 스케줄, 출퇴근, 시프트, **실시간 인건비 비율**
- **Inventory Management**: 재고 자동 차감, **레시피 관리**, **발주 알림**

### 2-3. Guest Engagement
- **Reservations**: 온라인 예약, 웨이팅, SMS/이메일
- **Gift Cards**: 디지털/물리 기프트카드
- **Marketing**: 이메일·앱 메시지, 프로모션, **세그먼트**, **CRM**
- **Loyalty**: 로열티 프로그램, 브랜드 웹앱, 리포트
- **Online Ordering**: 웹사이트 픽업·배달 주문 (커미션 $0)

### 3-2. 부분 구현 또는 미흡

| 블루프린트 항목 | 현재 상태 | 보완 필요 |
|-----------------|-----------|-----------|
| **리포팅** | admin_revenue, admin_hq_report 등 있으나 order_sessions 등 스키마 의존 | 메뉴별 수익성, 테이블 구성, 서버 생산성 등 **ad hoc 쿼리·대시보드** 강화 |
| **결제 수단** | 포인트/쿠폰만, 실제 카드·현금 결제 연동 없음 | PG 연동 또는 최소 **결제 수단 타입(현금/카드/기타) 기록** |
| **선물권(기프트카드)** | 없음 | 기프트카드 발급·잔액·사용 플로우 |
| **온라인 예약** | 웨이팅만, 예약(날짜·시간·인원) 없음 | 예약 테이블·플로우 |
| **마케팅** | 없음 | 이메일/SMS 캠페인, 프로모션, 세그먼트 |
| **로열티** | 포인트·쿠폰으로 일부 대체 | 구매 이력 기반 등급·리워드 자동화 |
| **재고 관리** | 메뉴 current_stock 차감만 | **레시피(메뉴→재료)**, **발주 알림**, 재고 리포트 |
| **직원 스케줄** | 계정·권한만 | **스케줄**, **출퇴근**, **실시간 인건비** |
| **체크 분할** | 없음 | split by item / by guests |
| **Customer Facing Display** | 없음 | 카운터용 고객 대면 화면 |
| **외부 연동** | store_delivery_api 등 일부 | 배민·쿠팡이츠 등 **배달앱 API**, 회계·페이롤 연동 |

## 4. POS 개발 경험 기반 권장사항

### 5-1. 단계별 우선순위

1. **1단계 (핵심)**
   - ~~결제 수단 타입 기록~~ ✅ 완료 (orders.payment_method, order_review 선택, store_orders 표시)
   - ~~리포팅 강화~~ ✅ 완료 (admin_revenue, admin_store_stats: 기간 필터, 채널별·결제수단별·메뉴별)
   - 기프트카드 기본 플로우 (발급·잔액·사용)

2. **2단계 (고객 경험)**  
   - 온라인 예약  
   - 체크 분할  
   - 로열티 등급·리워드 자동화

3. **3단계 (운영 효율)**  
   - 재고 레시피·발주 알림  
   - 직원 스케줄·출퇴근  
   - 마케팅(이메일/SMS, 프로모션)

4. **4단계 (연동)**  
   - PG 연동  
   - 배달앱 API  
   - 회계·페이롤 연동

### 5-2. DB·코드 원칙

- **기존 테이블·코드 삭제 금지**, **확장** 우선
- DB 스키마 변경 전 **반드시 확인 후 진행**
- API 응답은 **일관된 JSON 형식** 유지

### 5-3. UI·기기

- **하드웨어_기기_정의.md** 준수: 본사/가맹점(패드), 기사(휴대폰), 고객(반응형)
- **파스탈 톤** 유지, 터치 타겟 최소 44px
