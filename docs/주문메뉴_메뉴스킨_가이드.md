# 주문 메뉴 형식 & 메뉴 스킨 개발 가이드

## 1. 메뉴 스킨이란

- **메뉴 스킨** = 가맹점 업종에 맞는 **주문 메뉴 형식(구조)** 를 하나의 옵션으로 제공하는 개념입니다.
- 한식당, 커피숍, 피자, 드라이브스루 등 **업종별로 카테고리·메뉴·옵션(토핑, 굽기, 사이즈 등)** 구조가 정해진 것을 **스킨(포맷)** 으로 관리합니다.
- 어드민에서 **가맹점 신규 접수 시 업체 유형에 따라 메뉴 스킨을 선택해 제공**합니다.

---

## 2. 주문 메뉴 형식(구조)을 개발·수정하는 위치

| 구분 | 파일 | 역할 |
|------|------|------|
| **주문 타입 선택** | `index.php` | 매장 식사/포장/배달 선택, `store_id` 세션 저장 |
| **메뉴 목록(구조)** | `menu.php` | 카테고리·메뉴 로드, **stores.menu_format_id** 기준으로 포맷(스킨) 적용 |
| **메뉴 상세·옵션** | `menu_detail.php` | 메뉴별 옵션 그룹(토핑, 굽기, 사이즈 등) 표시 및 장바구니 담기 |
| **주문 완료** | `order_complete.php` | 주문·결제 처리, 옵션 텍스트 저장 |
| **진입점** | `index.php?store_id=N` | QR/링크로 매장 진입 시 `store_id` 설정 |

### 데이터 흐름

1. 고객이 **index.php?store_id=1** 등으로 진입 → `$_SESSION['store_id']` 설정
2. **menu.php**에서 `stores.menu_format_id` 조회 → 해당 포맷(스킨)의 **categories**, **menus** 로드
3. **menu_detail.php**에서 메뉴별 **option_groups** · **option_items** (추가 토핑, 굽기 등) 표시
4. 따라서 **메뉴 스킨 = menu_formats 한 건**과 그에 연결된 카테고리·메뉴·옵션 그룹입니다.

## 3. 메뉴 스킨을 옵션으로 제공하는 곳 (어드민)

| 위치 | 파일 | 내용 |
|------|------|------|
| **가맹점 관리** | `admin_store_manage.php` | 가맹점 신규/수정 시 **메뉴 스킨** 선택 (실제 저장: `stores.menu_format_id`) |
| **스킨(포맷) 목록·추가** | `admin_menu_format_list.php` | 제공할 스킨(한식당, 커피숍 등) 목록 및 일괄 추가 |
| **스킨(포맷) 편집** | `admin_menu_format_edit.php` | 스킨 추가/수정, **템플릿으로 포맷 생성** (카테고리+옵션 그룹 일괄 생성) |
| **포맷별 카테고리·메뉴** | `admin_menu_by_format.php` | 해당 스킨(포맷)의 카테고리·메뉴 관리 |
| **옵션 그룹 관리** | `admin_option_manage.php` | 옵션 그룹(토핑, 굽기, 사이즈 등) 및 항목 추가·가격 |

- **신규 가맹점 접수 시**: 어드민 → 가맹점 관리 → 해당 업체에 맞는 **메뉴 스킨** 선택 후 저장하면, 해당 가맹점의 주문 메뉴가 그 스킨(포맷) 구조로 노출됩니다.

---

## 4. DB 구조 (요약)

| 테이블 | 설명 |
|--------|------|
| `menu_formats` | 메뉴 스킨(업종) 정의 — 한식당, 커피숍, 피자 등 |
| `stores.menu_format_id` | 가맹점에 할당된 메뉴 스킨 ID |
| `categories` | 포맷별 카테고리 (menu_format_id) |
| `menus` | 포맷별 메뉴 (menu_format_id, category_id) |
| `option_groups` | 포맷별 옵션 그룹 (토핑, 굽기, 사이즈 등) |
| `option_items` | 그룹별 옵션 항목 (가격 포함) |
| `menu_option_groups` | 메뉴 ↔ 옵션 그룹 연결 |

---

## 5. 스킨별 UI를 다르게 하고 싶을 때

- 현재는 **모든 스킨이 같은 menu.php / menu_detail.php 레이아웃**을 사용합니다.
- 업종별로 **화면 레이아웃(카드형, 리스트형, 커피 전용 등)** 을 나누려면:
  1. `stores` 테이블에 `menu_skin_theme`(예: `default`, `coffee`, `drive`) 같은 컬럼을 추가하거나,
  2. `menu_formats`에 `layout_key` 컬럼을 두고,
  3. **menu.php** / **menu_detail.php**에서 `menu_format_id` 또는 `layout_key`에 따라 다른 HTML/ CSS 또는 include 파일을 로드하도록 분기하면 됩니다.

예시 (menu.php 내):

```php
$layout = 'default'; // 또는 stores.menu_skin_theme, menu_formats.layout_key 등
include __DIR__ . "/skins/menu_{$layout}.php";
```
