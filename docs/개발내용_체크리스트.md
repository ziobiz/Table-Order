# 식당용 다기능 POS — 개발 내용 및 설명 (체크리스트)

아래는 현재까지 개발된 기능과 설명입니다. 사이트에서 확인할 때는 링크를 클릭해 보시면 됩니다.

---

## 1. 고객 주문 시작 (진입점)

### index.php — 주문 방식 선택
- **역할**: QR 또는 URL로 접속 시 매장·테이블 정보(`store_id`, `table_no`)를 세션에 저장하고, 주문 방식을 선택하게 합니다.
- **기능**:  
  - 매장 식사(dine-in) / 픽업(pickup) / 배달(delivery) 중 선택  
  - `store_id`, `table_no` GET 파라미터로 매장·테이블 인식 (QR 연동)
- **확인 URL**: https://wirexcard.net/index.php  
  - 테이블 지정: https://wirexcard.net/index.php?store_id=1&table_no=A-1  

---

## 2. 메뉴·장바구니 (최근 정리 완료)

### menu.php — 메뉴 목록
- **역할**: 선택한 주문 방식(dine-in/pickup/delivery)과 언어에 따라 메뉴를 보여주고, 장바구니에 담긴 항목을 하단에 표시합니다.
- **기능**  
  - **6개 국어**: 한국어, English, Bahasa Indonesia, ไทย, 日本語, Tiếng Việt  
  - **3단 가격**: 매장/픽업/배달별 가격(`price`, `price_pickup`, `price_delivery`)  
  - **카테고리 필터**: "전체" + 카테고리 탭 클릭 시 해당 카테고리만 표시 (`category_id`)  
  - **가맹점 오버라이드**: 본사 메뉴 포맷 + 가맹점별 가격·노출 설정 반영  
  - **품절 표시**: `is_available`, `daily_limit`/`current_stock` 기반 품절 뱃지  
  - **장바구니**: 하단 주문 내역 리스트, 항목별 삭제(취소), 총 수량·총액  
  - **리뷰**: 메뉴 리스트 하단에 최근 리뷰 5개 **한 번만** 표시  
- **UI**: 심플 파스탈 톤 (배경 sky-50, 카드 border-sky-100, 버튼 sky-200/300)
- **확인 URL**: https://wirexcard.net/menu.php?order_type=dinein&lang=ko  

### menu_detail.php — 메뉴 상세·옵션·담기
- **역할**: 메뉴 하나를 상세 보여주고, 옵션 선택·수량 선택 후 장바구니에 담습니다.
- **기능**  
  - 메뉴 이미지·이름·설명·가격(주문 방식별)  
  - **상품 수량**: 1, 2, 3… 선택 (장바구니에 그 수량만큼 반영)  
  - **옵션 그룹**: 필수/선택 옵션, 옵션별 수량·가격  
  - 잘못된 메뉴 `id` 접근 시 메뉴 목록으로 리다이렉트  
- **UI**: 파스탈 톤, 상단 이미지 영역·옵션 카드·하단 장바구니 버튼 통일
- **확인 URL**: https://wirexcard.net/menu_detail.php?id=1&order_type=dinein&lang=ko  

### cart_process.php — 장바구니 담기 API
- **역할**: 메뉴 상세에서 "장바구니 담기" 시 호출되는 백엔드. 세션 장바구니에 추가합니다.
- **기능**  
  - `menu_id`, `order_type`, `quantity`, `options`(옵션 id·가격·수량) 수신  
  - 같은 메뉴+같은 옵션 조합이면 **수량만 가산**, 새 조합이면 **새 항목으로 추가**  
  - 응답: `{ "status": "success", "cart_count": N }` (일관된 JSON)
- **참고**: POST 전용. 브라우저에서 직접 열면 동작하지 않음.

## 3. 주문 확인·결제·완료

### order_review.php — 주문 확인(체크아웃)
- **역할**: 장바구니 내용을 최종 확인하고, 포인트 사용·게스트 정보 입력 후 결제 요청으로 넘깁니다.
- **기능**  
  - 장바구니 항목·수량·금액 표시, 항목별 삭제  
  - 사용 가능 포인트(GLOBAL) 표시·사용 금액 입력  
  - 비회원/배달용: 게스트 이름, 주소, 전화번호  
  - "추가 주문하기" 시 메뉴로 돌아갈 때 기존 `order_type` 유지  
- **확인 URL**: (장바구니에 담긴 뒤) https://wirexcard.net/order_review.php  

### order_complete.php — 주문 저장·포인트 적립
- **역할**: 주문 정보(주소·전화 등)를 DB에 저장하고, 포인트 정책에 따라 적립합니다.
- **기능**  
  - 주문 마스터(`orders`)·주문 상세(`order_items`) 저장  
  - 매장 포인트 정책(min_use_amount, max_use_pct 등)에 따른 포인트 사용·적립  
  - 재고 차감(`menus.current_stock`)  
  - 테이블 번호를 게스트 이름에 포함 (매장 식사 시)

## 4. 주방·가맹점·본사

### kitchen_display.php — KDS(주방 디스플레이)
- **역할**: 가맹점별 주문을 주방에서 볼 수 있는 화면.
- **기능**: store_id 기준 주문 목록, 테마/알림 설정, 조리 완료 등 상태 연동.
- **확인 URL**: https://wirexcard.net/kitchen_display.php?store_id=1  

### store_*.php (가맹점)
- **store_dashboard.php**: 가맹점 대시보드  
- **store_orders.php**: 주문 목록  
- **store_menu_list.php / store_menu_edit.php / store_menu_add.php**: 메뉴 목록·수정·추가  
- **store_category_manage.php**: 카테고리 관리  
- **store_qr_manage.php**: QR 관리  
- **store_setting.php**: 매장 설정 (KDS 테마 등)  
- **store_pos.php**: POS — 고객 QR 스캔 후 포인트 적립/사용  
- **store_buy.php**: 본사에서 포인트/쿠폰 재고 구매  
- **store_settlement.php**: 정산 요청  

### admin_*.php (본사)
- **admin_dashboard.php**: 본사 대시보드 (Store Manage, 메뉴 스킨, Policy, Sales, Settlement, Ledger 등 진입)  
- **admin_store_manage.php**: 가맹점 등록·수정·정책·메뉴 포맷 할당  
- **admin_menu_format_list.php / admin_menu_format_edit.php**: 메뉴 스킨(포맷) 관리  
- **admin_category_manage.php / admin_category_edit.php**: 카테고리 관리  
- **admin_menu_*.php**: 메뉴·옵션 관리  
- **admin_policy_manage.php**: 6대 정책 템플릿  
- **admin_sales.php**: 가맹점 포인트/쿠폰 구매 승인  
- **admin_settlement.php**: 정산 지급  
- **admin_ledger.php**: 거래(트랜잭션) 로그  
- **admin_staff_manage.php**: 직원·관리자 계정 관리  
- **admin_revenue.php / admin_hq_report.php 등**: 매출·리포트

## 6. 프로젝트 규칙 (Cursor 규칙)

- **pos-pastel-direction.mdc**: UI는 심플 파스탈 톤, POS 블루프린트 참조(1번 보안 제외), 기존 기능 유지·확장 우선, DB 변경 전 확인.  
- **deploy-and-delivery.mdc**: 수정/개발 후 **업로드할 파일 목록** 맨 마지막 안내, URL은 wirexcard.net 접두어, DB는 복붙 가능한 SQL로 제공, 배포는 FTP 덮어쓰기.

## 체크 시 확인 순서 제안

1. https://wirexcard.net/index.php — 주문 방식 선택  
2. https://wirexcard.net/menu.php?order_type=dinein&lang=ko — 메뉴·카테고리 필터·리뷰 한 번만  
3. 메뉴 하나 클릭 → https://wirexcard.net/menu_detail.php?id=1&order_type=dinein&lang=ko — 수량·옵션 후 담기  
4. 하단 "주문완료 하기" → order_review → order_complete 까지 흐름 확인  
5. (관리자) 로그인 후 admin_dashboard, store 쪽 필요한 메뉴 확인

*이 문서는 개발 내용 체크용입니다. 실제 배포 시에는 수정된 PHP 파일만 FTP로 업로드하면 됩니다.*
