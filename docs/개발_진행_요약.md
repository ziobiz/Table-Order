# 개발 진행 요약 (저장)

## 1. KDS ↔ store_orders 연동
- **kitchen_display.php**: 조리 완료 시 `kds_sync_order_status`(연동함/연동 안 함) 반영, `orders.status`는 연동함일 때만 SERVED, `order_items.item_status` = SERVED, 연동 실패 시 sync_fail 메시지

## 2. 메뉴 5종 시드
- **seed_5_menus.php**: 김치찌개, 비빔밥, 삼겹살, 된장찌개, 제육볶음 생성
- **migrations/seed_5_food_menus.sql**: 동일 내용 SQL

## 3. 메뉴 포맷(메뉴 스킨) 15종
- **admin_menu_format_list.php**: 미리 정의 포맷 15종 일괄 추가 (한식당, 양식당, 피자, 마라탕, 중국, 마사지, 약국, 일본스시, 비어가든, 스테이크, 부페, **커피숍, 베이커리샵, 드라이브스루, 샤브샤브점**)
- **admin_menu_format_edit.php**: 리스트별 템플릿으로 포맷+카테고리+**옵션 그룹**(토핑, 굽기, 사이즈 등) 일괄 생성
- **migrations/seed_menu_formats_11.sql**: 15종 포맷 INSERT SQL

## 4. 메뉴 스킨 = Store Manage에서 할당
- **admin_store_manage.php**: 가맹점 등록/수정 시 **메뉴 스킨** 선택 (stores.menu_format_id), 설명 문구 통일
- **admin_dashboard.php**: Store Manage·메뉴 스킨 카드 문구 정리
- **admin_menu_format_list.php**: 제목을 "메뉴 스킨"으로 통일
- **docs/주문메뉴_메뉴스킨_가이드.md**: 주문 메뉴 형식/스킨 구조·개발 위치·어드민 역할 정리

## 5. 옵션 있는 메뉴(피자 등) 만들기
- **admin_menu_edit.php**: 메뉴 편집 시 **옵션 그룹** 선택(토핑, 사이즈, 굽기 등) 연결 저장, 수정 시 포맷별 옵션 그룹 목록 표시

## 6. stores 저장 오류 수정
- **admin_store_manage.php**: 저장 쿼리에서 `currency` → `currency_code` 사용
- **migrations/add_stores_owner_manager_policy_columns.sql**: owner_name, owner_tel, manager_*, biz_no, tax_*, address, tel, 정책·쿠폰 관련 컬럼 등 stores 테이블에 추가
