# 식당용 다기능 POS — 업무 구조 정리

---

## 1. 가맹점 등록 및 계정 오픈 (본사)

가맹점이 **신규 가맹점**으로 등록되면, **본사**에서 본사 페이지를 통해 가맹점을 생성하고 **계정을 오픈**해 이용할 수 있게 한다.

### 1-1. 가맹점 신청 방식 (2가지)

| 구분 | 경로 | 본사 처리 |
|------|------|-----------|
| **오프라인** | 연락 → 신청서 작성 → 가맹점 등록 요청 | 본사 계정으로 로그인 후 **가맹점 정보를 직접 입력**하고, **가맹점 계정을 생성**하여 오픈 |
| **온라인** | 가맹점이 웹에서 **온라인 가맹점 등록 신청** | 신청 내용 확인 후 **본사 승인** → 승인 시 **가맹점 계정 활성화** |

### 1-2. 정리

- **오프라인**: 본사가 가맹점 정보 입력 + 계정 생성 (즉시 오픈).
- **온라인**: 가맹점이 스스로 신청 → 본사 승인 → 계정 활성화.

## 2. 가맹점 대시보드 — 솔루션 설정 (판매 준비)

가맹점은 **본인 사이트에 접속**하여 **대시보드**에서 아래 설정을 직접 수행한다.  
설정이 완료되면 **판매 준비**가 된 것으로 본다.

### 2-1. 메뉴·가격

| 항목 | 설명 |
|------|------|
| 메뉴 구성 | 매장에서 판매하는 메뉴 정의 (카테고리, 메뉴, 옵션 등) |
| 외부형 메뉴 구성 | 외부 채널용 메뉴 구조 (형식/노출용) |
| 가격 구성 | 매장/픽업/배달 등 판매 채널별 가격 설정 |
| 외부 가격 구성 | **외부 솔루션 연동용** — 배민·타 배달앱 등에 노출하기 위한 가격 구성 |

### 2-2. 결제·지원·정책

| 항목 | 설명 |
|------|------|
| 결제 구성 | 카드 등 결제 방식에 대한 세팅 |
| 지원 구성 | 결제/기타 지원 관련 설정 |
| 쿠폰 정책 | 쿠폰 발급·사용 규칙 |
| 포인트 정책 | 포인트 적립·사용 규칙 |
| 소셜 연동 | SNS/소셜 로그인 등 연동 |

### 2-3. 매장 운영 도구

| 항목 | 설명 |
|------|------|
| 테이블 QR 생성 | 테이블별 주문용 QR 코드 생성·관리 |
| 주방 디스플레이 세팅 | KDS(키친 디스플레이) 테마·알림 등 설정 |
| 외부 배달 APP 연동 | 배달 앱과의 **API 연동** 및 배달 시스템 구성 |

### 2-4. 인력·배달·매출·예약

| 항목 | 설명 |
|------|------|
| 직원 관리 | 직원 계정·권한·역할 관리 |
| 배달 관리 | **자체 배달 기사** 관리, **외부 배달 기사** 관리 |
| 매출 관리 | 일/주/월 매출 조회·리포트 |
| 외상 관리 | 외상(후불) 거래 관리 |
| 예약 관리 | **온라인 예약** 관리 + **식당 앞 방문 대기** 시스템 |

## 3. 고객 이용 구조 (3가지 이용방법 × 2가지 고객 타입)

### 3-1. 이용 방법 (3가지)

| 구분 | 설명 |
|------|------|
| **매장 방문** | 식당 내 방문·테이블 식사 (dine-in) |
| **픽업** | 고객이 직접 매장에서 수령 (pickup) |
| **배달** | 매장/자체·외부 기사가 배달 (delivery) |

### 3-2. 고객 타입 (2가지)

| 구분 | 우선순위 | 비고 |
|------|----------|------|
| **비회원** | **1차 구축** | 대부분이 비회원이므로, 비회원 플로우를 먼저 구축 |
| **회원** | 2차 구축 | 회원 전용 페이지를 두고, 그 안에서 이용 가능하도록 구축 |

### 3-3. 정리

- **비회원**: 주문·결제·픽업/배달까지 비회원 기준으로 먼저 완성.
- **회원**: 이후 회원 전용 페이지에서 로그인 후 동일한 3가지 이용방법(매장/픽업/배달) 이용.

## 4. 배달 기사 관리 (자체 배달 기사)

### 4-1. 개념

- 식당이 **직접 등록한 배달 기사**를 사용하는 방식.
- 예: 식당 근처 **오토바이 배달 기사**를 등록해, 해당 기사가 배달을 수행하도록 서비스 제공.

### 4-2. 배달 기사 화면 요구사항

- **배달 기사로 로그인**하면, **배달 기사가 봐야 할 정보만** 노출된다.
- 화면·기능 기준은 **타 업체(배달 앱/배달 관리 솔루션)의 배달 기사 앱/화면 기준**을 그대로 적용한다.

### 4-3. 타 업체 기준 적용 시 포함될 typically 내용 (참고)

- 배달 할당 목록 (신규/진행중/완료)
- 주문 상세 (주소, 전화, 메뉴, 금액, 요청사항)
- 수락/거절, 출발/도착/완료 등 상태 변경
- 내 배달 이력·정산 요약

*(실제 구현 시 DB·API 설계 전에 “배달기사 화면에 필요한 항목”을 한 번 더 정리하는 것을 권장)*

## 5. 업무 흐름 요약 (단계별)

```
[1] 가맹점 등록
    ├─ 오프라인: 신청서 → 본사가 정보 입력·계정 생성 → 계정 오픈
    └─ 온라인: 가맹점 온라인 신청 → 본사 승인 → 계정 활성화

[2] 가맹점 대시보드 설정 (판매 준비)
    ├─ 메뉴·가격 (자체 + 외부 연동용)
    ├─ 결제·쿠폰·포인트·소셜
    ├─ 테이블 QR, KDS, 외부 배달 APP API
    ├─ 직원·배달·매출·외상·예약(온라인+대기)
    └─ 설정 완료 → 판매 준비 완료

[3] 고객 이용
    ├─ 비회원 우선: 매장/픽업/배달 주문·결제
    └─ 회원: 회원 전용 페이지에서 매장/픽업/배달 이용

[4] 배달 기사
    └─ 식당 등록 기사 로그인 → 타 업체 기준에 맞는 배달 기사용 화면 노출
```

| 본사 가맹점 생성 | admin_store_manage.php 등 |

| 쿠폰·포인트 정책 | store 설정, point_engine, policy |
| 직원 관리 | admin_staff_manage |
| 매출 관리 | store_orders, admin_revenue 등 |
| 고객: 매장/픽업/배달, 비회원 | index → menu → order_review → order_complete |

| 배달 기사 전용 화면 | 별도 구축 대상 (타 업체 기준 적용) |
